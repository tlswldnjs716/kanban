# PRD (Product Requirements Document) — 칸반보드

> 근거 문서: `plan.md`

## 1. 개요
HTML/CSS/JS만으로 동작하는 단일 사용자용 정적 칸반보드. To-do / In-progress / Done 3개 컬럼으로 작업을 시각화하고, 드래그 앤 드롭으로 작업 진행 상태를 바꿀 수 있게 한다.

## 2. 배경 및 목적
- 별도 백엔드 없이 브라우저에서 바로 동작하는 가벼운 작업 관리 도구가 필요하다.
- HTML/CSS/JS를 파일 단위로 분리하는 학습 목적의 예제이기도 하다.

## 3. 대상 사용자
- 개인 사용자 1인 기준 (로그인/멀티유저 없음, 같은 브라우저의 localStorage를 공유하는 단일 사용자 가정).

## 4. 범위 (Scope)

### 4.1 In-Scope (필수 기능)
| ID | 기능 | 설명 |
|---|---|---|
| F-1 | 3개 컬럼 표시 | To-do, In-progress, Done 컬럼을 항상 화면에 표시 |
| F-2 | 카드 추가 | 각 컬럼에서 텍스트를 입력해 새 카드를 추가 |
| F-3 | 카드 삭제 | 카드의 삭제 버튼으로 개별 카드 제거 |
| F-4 | 드래그 앤 드롭 이동 | 카드를 드래그해서 다른 컬럼으로 이동 (HTML5 Drag and Drop API) |
| F-5 | 상태 영속화 | 카드 내용과 위치를 localStorage에 저장하고 새로고침 후에도 복원 |

### 4.2 Out-of-Scope (이번 범위 제외)
- 사용자 인증/로그인, 멀티유저 동시 편집
- 서버/DB 연동 (단, `DATABASE_DESIGN.md`에서 향후 RDB 전환 여지를 설계해 둠)
- 카드 내 세부 항목(담당자, 마감일, 라벨 등), 컬럼 추가/삭제, 카드 정렬 순서 변경(같은 컬럼 내 위치 이동)
- 실행 취소(undo), 검색/필터, 다국어

## 5. 핵심 사용자 시나리오
1. 사용자가 페이지를 열면 이전에 저장된 카드들이 컬럼별로 그대로 보인다.
2. 사용자가 To-do 컬럼에 새 작업을 입력해 카드를 추가한다.
3. 작업을 시작하면 카드를 To-do에서 In-progress로 드래그한다.
4. 작업이 끝나면 카드를 In-progress에서 Done으로 드래그한다.
5. 더 이상 필요 없는 카드는 삭제 버튼으로 제거한다.
6. 새로고침해도 위 상태가 그대로 유지된다.

## 6. 성공 기준 (Acceptance Criteria)
- [x] 3개 컬럼이 항상 표시되고 각 컬럼에 카드 개수가 보인다.
- [x] 카드 추가/삭제가 즉시 화면에 반영된다.
- [x] 카드를 다른 컬럼에 드롭하면 해당 컬럼으로 이동하고 원래 컬럼에서는 사라진다.
- [x] 새로고침 후에도 카드 텍스트와 컬럼 위치가 동일하게 복원된다.
- [x] 콘솔 에러 없이 동작한다.

> 위 기준은 `TASKS.md` 7번 섹션 통합 검증을 통해 사용자가 브라우저에서 직접 확인 완료함 (이상 없음).

## 7. 비기능 요구사항
- 외부 라이브러리/프레임워크 없이 순수 HTML/CSS/JS로 구현 (빌드 단계 없음).
- 모던 데스크톱 브라우저(Chrome/Edge/Firefox 최신 버전) 기준 동작.
- 모바일 화면에서도 컬럼이 깨지지 않고 세로로 보이도록 반응형 처리 (터치 드래그는 이번 범위에서 보장하지 않음, `TASKS.md` 참고).

## 8. 향후 고려 사항 (이번 범위 아님, 확장 여지)
- 백엔드 DBMS(MySQL/PostgreSQL) 연동을 통한 다중 기기/다중 사용자 동기화 — `DATABASE_DESIGN.md`에 스키마 설계 선반영.
- 컬럼 내 카드 순서 변경, 컬럼 자체 추가/이름 변경.
