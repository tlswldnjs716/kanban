# Design System — 칸반보드

> 근거 문서: `plan.md`

## 1. 레이아웃
- 보드(`.board`)는 flex 컨테이너로 3개 컬럼을 가로로 배치한다.
- 데스크톱: 3컬럼 가로 배치, 컬럼은 동일한 너비로 균등 분할.
- 모바일(좁은 화면): 컬럼을 세로로 스택해 한 화면에 한 컬럼씩 보이도록 반응형 전환.
- 브레이크포인트 기준: `768px` (이하 모바일, 초과 데스크톱).

## 2. 컬러 팔레트
| 용도 | 컬럼 | 색상(예시) | 설명 |
|---|---|---|---|
| 컬럼 헤더 | To-do | `#64748b` 계열 (회색-블루) | 아직 시작 전, 중립색 |
| 컬럼 헤더 | In-progress | `#f59e0b` 계열 (주황/앰버) | 진행 중, 주의를 끄는 색 |
| 컬럼 헤더 | Done | `#16a34a` 계열 (초록) | 완료, 긍정적인 색 |
| 배경 | 전체 페이지 | `#f3f4f6` (밝은 회색) | 컬럼/카드와 대비되는 배경 |
| 카드 배경 | 공통 | `#ffffff` (흰색) | 컬럼 배경과 구분되는 카드 표면 |
| 텍스트 | 본문 | `#1f2937` (다크 그레이) | 가독성 좋은 본문 색 |
| 강조/위험 | 삭제 버튼 | `#ef4444` (레드) | 삭제는 위험 동작임을 시각적으로 표시 |

컬럼별 헤더 색상은 위 3색을 기준으로 구분하되, 정확한 헥스값은 구현 시 `style.css`에서 CSS 변수(`--color-todo`, `--color-inprogress`, `--color-done` 등)로 한 곳에 모아 관리한다.

## 3. 타이포그래피
- 시스템 기본 sans-serif 폰트 스택 사용 (`-apple-system, "Segoe UI", Roboto, sans-serif` 등). 별도 웹폰트 로드하지 않음 — 외부 의존성 없는 정적 페이지 원칙 유지.
- 컬럼 제목: 16~18px, bold.
- 카드 텍스트: 14~15px, normal.
- 카드 개수 배지: 12~13px, 보조 정보로 톤 다운.

## 4. 컴포넌트

### 4.1 컬럼 (`.column`)
- 둥근 모서리(`border-radius: 8px`), 연한 배경, 내부 패딩.
- 구성: 헤더(제목 + 카드 개수) → 카드 추가 폼(입력창 + 버튼) → 카드 리스트(`.card-list`, 세로 스크롤 가능).

### 4.2 카드 (`.card`)
- 흰 배경, 옅은 그림자(`box-shadow`)로 컬럼 배경과 분리.
- `draggable="true"` 상태에서 `cursor: grab`, 드래그 중에는 `cursor: grabbing`.
- 우측 또는 하단에 삭제(×) 버튼 배치.
- 상태 클래스:
  - `.dragging`: 드래그 중인 카드 — `opacity: 0.5` 등으로 반투명 처리.
  - (호버) 삭제 버튼이 살짝 강조되는 정도의 미세 인터랙션.

### 4.3 드롭 영역 강조 (`.drag-over`)
- 카드를 컬럼 위로 드래그하고 있을 때 해당 컬럼의 `.card-list`에 적용.
- 시각적으로 점선 테두리(`outline: 2px dashed`) 또는 배경색 변화로 "여기에 놓을 수 있다"를 명확히 표시.

### 4.4 카드 추가 폼
- 각 컬럼 하단(또는 상단)에 텍스트 입력창 + "추가" 버튼.
- placeholder로 입력 가능함을 안내 (예: "새 카드 추가...").
- 버튼은 컬럼 강조색과 통일감 있게, 또는 중립색(회색/블루)으로 단순화.

### 4.5 카드 개수 배지
- 컬럼 헤더 옆에 작은 원형/캡슐 배지로 현재 카드 수 표시 (예: `3`).

## 5. 인터랙션/모션
- 카드 추가/삭제 시 별도 애니메이션 없이 즉시 반영 (단순함 우선, 트랜지션은 선택적으로 `opacity`/`transform` 0.15~0.2s 정도의 짧은 효과만 고려).
- 드래그 중 카드: `.dragging` 클래스로 반투명 처리.
- 드롭 가능 영역: `.drag-over` 클래스로 테두리/배경 강조, 드롭 또는 드래그 종료 시 즉시 해제.

## 6. 반응형 규칙
- `>= 768px`: `.board { display: flex; flex-direction: row; }`, 컬럼 너비 균등.
- `< 768px`: `.board { display: flex; flex-direction: column; }`, 컬럼은 전체 너비 사용, 위아래로 스크롤.

## 7. 접근성 메모
- 카드 추가 입력창에 `placeholder`와 함께 의미 있는 `aria-label` 또는 연결된 `<label>` 제공.
- 삭제 버튼에 `aria-label="카드 삭제"` 등 스크린리더용 텍스트 제공 (아이콘/× 문자만으로는 의미 전달 부족).
- 색상만으로 컬럼을 구분하지 않고 텍스트 제목(To-do/In-progress/Done)을 항상 함께 표시 (이미 기본 설계에 포함됨).
