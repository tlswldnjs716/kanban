# User Flow — 칸반보드

> 근거 문서: `plan.md`, `PRD.md`

## 1. 진입 플로우 (페이지 로드)
```
페이지 접속
  └─ localStorage에 저장된 상태 있음?
       ├─ 있음 → 상태를 읽어 3개 컬럼에 카드 렌더링
       └─ 없음/손상됨 → 빈 보드(3개 컬럼, 카드 없음)로 시작
```

## 2. 카드 추가 플로우
```
사용자가 컬럼 하단의 입력창에 텍스트 입력
  └─ "추가" 버튼 클릭 또는 Enter
       ├─ 입력값이 공백/빈 문자열 → 아무 동작 안 함(폼 제출 막힘)
       └─ 유효한 텍스트
            ├─ 새 카드 객체 생성 (id, text)
            ├─ 해당 컬럼 배열 끝에 추가
            ├─ localStorage에 저장
            ├─ 해당 컬럼 다시 렌더링 (새 카드 표시)
            └─ 입력창 비움 (다음 입력 대기)
```

## 3. 카드 삭제 플로우
```
사용자가 카드의 삭제(×) 버튼 클릭
  └─ 해당 카드 id를 상태에서 검색
       └─ 상태(어느 컬럼이든)에서 카드 제거
            ├─ localStorage에 저장
            └─ 보드 다시 렌더링 (카드 사라짐)
```

## 4. 드래그 앤 드롭(컬럼 간 이동) 플로우
```
사용자가 카드를 마우스로 누르고 드래그 시작 (dragstart)
  ├─ 카드에 dragging 스타일 적용 (반투명 등)
  └─ 드래그 중인 카드의 id를 dataTransfer에 저장

사용자가 다른 컬럼의 카드 리스트 위로 드래그 (dragover)
  ├─ 기본 동작 막기 (preventDefault) → drop 허용
  └─ 해당 컬럼에 drag-over 스타일 적용 (드롭 가능 영역 강조)

사용자가 컬럼 영역에서 마우스를 뗌 (drop)
  ├─ dataTransfer에서 카드 id 읽기
  ├─ 기존 컬럼 배열에서 해당 카드 제거
  ├─ 대상 컬럼 배열 끝에 카드 추가
  ├─ localStorage에 저장
  ├─ 보드 다시 렌더링 (카드가 새 컬럼에 표시)
  └─ drag-over 스타일 제거

드래그가 컬럼 밖에서 끝남 (dragend, drop 없이 종료)
  └─ dragging 스타일만 원복, 상태 변경 없음
```

## 5. 새로고침 플로우
```
사용자가 브라우저 새로고침
  └─ "1. 진입 플로우"와 동일하게 localStorage 기준으로 보드 복원
       → 추가/삭제/이동했던 마지막 상태가 그대로 보임
```

## 6. 전체 사용자 여정 예시 (End-to-End)
1. 페이지 접속 → 빈 보드 확인.
2. To-do에 "기획서 작성" 카드 추가.
3. To-do에 "발표 자료 준비" 카드 추가.
4. "기획서 작성" 카드를 In-progress로 드래그.
5. 새로고침 → To-do에 "발표 자료 준비", In-progress에 "기획서 작성"이 그대로 보임.
6. "기획서 작성" 카드를 Done으로 드래그.
7. "발표 자료 준비" 카드를 삭제(취소된 작업).
8. 새로고침 → To-do는 비어 있고 Done에 "기획서 작성"만 남아 있음.
